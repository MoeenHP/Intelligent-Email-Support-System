## معرفی پروژه

این سیستم یک ایمیل خام مشتری را دریافت می‌کند و به‌صورت هوشمند:

1. تشخیص می‌دهد که ایمیل شامل چه درخواست‌هایی است
2. تصمیم می‌گیرد کدام دپارتمان‌ها (فروش، فنی، مالی) باید پاسخ‌گو باشند
3. اطلاعات لازم را از سرویس‌های داخلی جمع‌آوری می‌کند
4. یک ایمیل پاسخ یکپارچه و حرفه‌ای تولید می‌کند

**ویژگی‌های کلیدی:**
- کاملاً آفلاین — بدون نیاز به اینترنت یا API خارجی
- پیاده‌سازی الگوریتم‌های NLP (TF-IDF و Cosine Similarity) از صفر
- پشتیبانی از زبان فارسی و انگلیسی
- خروجی استاندارد JSON

## نحوه اجرا

### پیش‌نیازها

- Python 3.7 یا بالاتر
- هیچ کتابخانه خارجی نیاز نیست — فقط Python استاندارد

### نصب و راه‌اندازی

```bash
# کلون کردن ریپازیتوری
git clone https://github.com/your-username/email-support-system.git
cd email-support-system

# اجرا با ایمیل نمونه پیش‌فرض

# اجرا با ایمیل دلخواه
python main.py "سلام، سفارش 12345 دارم که نرسیده. لپتاپ X200 هم مشکل فنی دارد."
```

## توضیح لایه‌ها

### لایه ۱ — پیش‌پردازش (`preprocessor.py`)

متن خام ایمیل را برای پردازش آماده می‌کند.

| مرحله | عملکرد | مثال |
|---|---|---|
| نرمال‌سازی | یکسان‌سازی کاراکترهای فارسی | `ي` → `ی`، `ك` → `ک` |
| Lowercase | تبدیل انگلیسی به حروف کوچک | `X200` → `x200` |
| Tokenization | تقسیم متن به کلمات | `"سفارش دارم"` → `["سفارش", "دارم"]` |
| Stopword removal | حذف کلمات بی‌معنی | حذف `و، در، به، از، که` |

### لایه ۲ — مسیریاب (`router.py`)

هسته هوشمند سیستم. دو الگوریتم NLP کلاسیک را از صفر پیاده‌سازی می‌کند:

**TF-IDF (Term Frequency - Inverse Document Frequency):**
```
TF(t, d)  = تعداد تکرار کلمه در سند / کل کلمات سند
IDF(t, D) = log(تعداد کل اسناد / تعداد اسنادی که کلمه دارند)
TF-IDF    = TF × IDF
```

**Cosine Similarity:**
```
similarity(A, B) = (A · B) / (|A| × |B|)
```

هر دپارتمان یک "پروفایل" از کلمات کلیدی دارد. سیستم شباهت ایمیل را با هر پروفایل اندازه می‌گیرد و به دپارتمان‌هایی که بالای آستانه هستند مسیریابی می‌کند.

### لایه ۳ — استخراج موجودیت (`entity_extractor.py`)

با استفاده از Regex و قوانین دستی، داده‌های ساختاریافته را از متن استخراج می‌کند:
- **شماره سفارش:** اعداد ۴ تا ۱۲ رقمی نزدیک کلمات مربوط به سفارش
- **نام محصول:** تطبیق با لیست محصولات شناخته‌شده + الگوی شماره مدل
- **سیگنال مالی:** تشخیص کلمات بازگشت وجه، استرداد، کنسل
- **سیگنال فنی:** تشخیص کلمات مشکل فنی، خرابی، ارور

### لایه ۴ — هماهنگ‌ساز (`orchestrator.py`)

مغز تصمیم‌گیر سیستم. بر اساس نتایج لایه‌های ۲ و ۳:
1. یک برنامه تسک می‌سازد (کدام ابزار با چه پارامتری فراخوانی شود)
2. تسک‌ها را به‌ترتیب اجرا می‌کند
3. یک audit trail کامل از همه مراحل ثبت می‌کند

### لایه ۵ — ابزارها (`tools.py`)

سه API داخلی شرکت را شبیه‌سازی می‌کند:

| تابع | ورودی | خروجی |
|---|---|---|
| `get_order_status(order_id)` | شماره سفارش | وضعیت، کد رهگیری، زمان تحویل |
| `get_product_info(product_name)` | نام محصول | مشخصات فنی، لینک راهنما |
| `get_refund_policy()` | بدون ورودی | متن کامل سیاست بازگشت وجه |

### لایه ۶ — تولید پاسخ (`response_generator.py`)

نتایج ابزارها را به ایمیل تبدیل می‌کند:
- یک **موتور template** ساده که خودمان نوشتیم
- برای هر دپارتمان یک بخش جداگانه
- ادغام همه بخش‌ها در یک ایمیل منسجم
- تولید فایل JSON نهایی

## الگوهای طراحی

### ۱. Pipeline Pattern (الگوی خط لوله)
هر لایه یک تبدیل مشخص روی داده انجام می‌دهد و نتیجه را به لایه بعد می‌دهد. این الگو کد را ماژولار، قابل تست و قابل توسعه نگه می‌دارد.

### ۲. Strategy Pattern (الگوی استراتژی)
در `response_generator.py` از یک دیکشنری استفاده کردیم که نام دپارتمان را به تابع سازنده آن بخش map می‌کند. این اجازه می‌دهد دپارتمان جدید اضافه شود بدون تغییر در منطق اصلی.

### ۳. Dispatcher Pattern (الگوی توزیع‌کننده)
تابع `call_tool()` در `tools.py` یک نقطه ورودی واحد برای همه ابزارهاست. Orchestrator فقط با این dispatcher صحبت می‌کند، نه مستقیماً با توابع.

### ۴. Fallback Pattern (الگوی پشتیبان)
در `router.py`، اگر TF-IDF نتوانست مسیریابی کند، سیگنال‌های Entity Extraction به‌عنوان fallback وارد می‌شوند. این سیستم را در برابر ایمیل‌های کوتاه یا غیرمعمول مقاوم می‌کند.

## ساختار خروجی

خروجی سیستم یک فایل JSON با سه بخش است:

```json
{
  "original_text": "سلام، سفارش 78432 دارم...",

  "processing_steps": [
    {
      "step": 1,
      "action": "استخراج موجودیت‌ها",
      "result": {
        "order_ids": ["78432"],
        "product_names": ["لپتاپ X200"],
        "has_refund": true
      }
    },
    {
      "step": 2,
      "action": "مسیریابی به دپارتمان‌ها",
      "result": {
        "departments_identified": ["sales", "technical", "finance"]
      }
    }
    // ... مراحل بعدی
  ],

  "final_response": "سلام و احترام،\n\nبا تشکر از تماس شما..."
}
```
